# Agent Benchmark Results — 2026-07-21

**Model:** claude-sonnet-4-6

> First benchmark run. 3 tasks (polyflow corpus, impact cases), 3 arms, 1 trial each. Key findings: (1) Arm 1 (semantic) recalls 0.333 — two tasks returned no file paths in the agent answer, a presentation failure where the agent described callers verbally without listing paths. (2) Arm 2 (FTS-only) and arm 3 (no MCP) both recall 1.000, but arm 3 baseline is inflated: `claude -p` in Claude Code headless mode retains built-in file-search tools (Bash, Read) regardless of `--strict-mcp-config`, so arm 3 is not truly tool-free — a design limitation for future runs. (3) Precision is uniformly low (0.2–0.33) across all arms because the agent lists many more files than needed. Deviations from protocol: (a) 1 trial per task/arm instead of 3 (cost); (b) polyflow corpus only (3 tasks) instead of 10 per repo, because the MCP server reads from the current-directory DB and other repos would need per-repo invocation; (c) arm 3 baseline contaminated by Claude Code built-in tools.
